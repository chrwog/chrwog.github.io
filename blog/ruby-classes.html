Ruby Classes

The first example that comes to mind when thinking about how to explain classes and methods in ruby is a microwave. A microwave has many different functions. Most microwaves can cook food, defrost, tell time, and countdown from a given time. Some microwaves even have other completely different functions like over the range microwaves can exhaust steam from cooking on the stove! This is very similar to how classes and methods work.

You can think of the class as the microwave. It is the main body that everything else revolves around. The methods are like the functions of the microwave (cook, defrost, etc.)

Classes have variables that can represent just about anything. These are called instance variables. Keeping with our microwave analogy, an instance variable that a microwave would have is the time. When you want to cook something in the microwave you enter a time and then press cook. When you want to defrost something you enter the time and then hit defrost. The common variable is time. A lot of the microwave functions can do something with the time input. This is just like the methods in a class. Methods in the class can use the instance variable when running their code.

Methods also have their own personal variables that can only be used inside that method. This would be like selecting the specific power setting on the microwave. The microwave power function could have low medium or high. The cook and defrost functions can't use these variables. They can only use a time input. Below is what the code for a microwave class would look like

class Microwave
  def initialize
    @time = 4
  end
  
  def power (setting)
    @current_power = setting
  end
  
  def cook
    @time + @current_power
  end
  
  def defrost
    frost_power = @current_power - 4
  end
  
  def timer
    while @time > 0
    @time -= 1
  end
end
