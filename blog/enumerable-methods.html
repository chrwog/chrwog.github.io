Enumerable#map

12.13.15

The enumerable #map is an invaluable expression that I personally used for the group research methods challenge this week. More specifically I used the destructive version of this method.

We were given an array filled with strings and numbers, and a hash filled with string keys and number values: 

i_want_pets = ["I", "want", 3, "pets", "but", "only", "have", 2]
my_family_pets_ages = {"Evi" => 6, "Ditto" => 3, "Hoobie" => 3, "George" => 12, "Bogart" => 4, "Poly" => 4, "Annabelle" => 0}

My objective was to write a method that let the user give a number to increase the number of pets they wanted. Additionally another method had to be written to update the ages of the current pets in the hash.

The .map! came to the rescue!

The .map method returns a new array with the results of running the code block for every element. That sounds an awful like the .each method that many people use right? Well the key difference is that map returns the results of the iterations to a new object. The .each method will run the block, discard the results, and return the original object.

For the group challenge the map! method let me retrieve the proper results. Below is the code I used:

def my_array_modification_method!(source, thing_to_modify)
   source.map! do |x|  
     if x.is_a?(Integer)
       x += thing_to_modify
     else x
     end
   end
 end

The first step was to take the first element of the array and put it into variable x. I then checked to see if it was an integer with the .is_a? method. If it was, the user given number (thing_to_modify) was added to it. The .map! method lets the return be the result as well as change the original array element as per the instructions given.

I hope this example shows how .map can be used effectively. Happy coding!
