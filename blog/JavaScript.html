Ruby Hashes vs JavaScript Objects

Ruby Hashes and JavaScript Objects are very similar with a few key differences. They are both considered objects in their respective languages, and store data. The differences are in the details.

Lets start with what they have in common. In Ruby hashes store data as key-value pairs. The key represents the way a user would access the data associated with that key. In JavaScript you can do something similar by creating an object with properties associated with values. Below is an example of both. 


ruby_hash = {
  "Dog_name" => "Rex",
  "Cat_name" => "Snickers",
"Fish_name" => "Goldfinger"
  }


var javaScriptObject = {
  dogName: "Rex",
  catName: "Snickers",
  fishName: "Goldfinger"
};

You can see how they are similar but also slightly different in terms of the way it is written. Coding languages are like real world languages. Some languages are very similar, but have little nuances that make them different.

The way you add to them is different depending on the language. In ruby you can add to a hash this way:


ruby_hash["Hamster_name"] = "Turbo"

In JavaScript you do this:


javaScriptObject.hamsterName = "Turbo"

I hope this help you learn a little about how Ruby and JavaScript are both similar and different.
